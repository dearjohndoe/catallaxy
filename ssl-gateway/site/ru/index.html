§00 · THE GAP

Две половины рынка, которого пока нет.

ИИ-агенты умеют принимать решения, но не умеют платить. Разработчики умеют делать полезные вещи, но не умеют брать за них деньги. Catallaxy закрывает обе стороны сразу.

СТОРОНА СПРОСА

Агенты решают. Платить — не могут.

Ассистент понимает, что пользователю нужны Stars, VPN-конфиг или один вызов GPT, — и упирается в стену форм, карт и логинов, сделанных для людей.

Catallaxy даёт агентам способ платить и забирать результат — без человека.
СТОРОНА ПРЕДЛОЖЕНИЯ

Разработчики делают. Брать деньги — не могут.

У готового скрипта или API нет способа приносить деньги без юрлица, платёжного провайдера и категории, которую не заблокируют.

Catallaxy превращает любую программу в платный продукт за ~20 строк.
§01 · THE PROTOCOL

Минимальная основа.
Из неё собирается всё.

Без привязки к SDK и фреймворкам. Платёжная грамматика, слой обнаружения и контракт продукта в один файл — каждый работает сам по себе.

Агент — это пайп. Оплата — это статус-код.

402 handshake
$ POST https://agent.catallaxy.cc/invoke { "capability": "translate", "body": {…} } ← 402 Payment Required { "payment_options": [ { "rail":"TON", "amount":"150000000", "memo":"a91f…" }, { "rail":"USDT", "amount":"200000" } ] } # кошелёк подписывает и отправляет tx с memo $ POST /invoke { tx, memo } ← 200 OK { "status":"done", "result":{…} }
ПРИМИТИВ 01 — PAY

Оплата через HTTP 402

Оплата как статус-код. На запрос сервер отвечает ценой; клиент платит ончейн и повторяет запрос с доказательством.

← 402 Payment Required
  rail:  TON | USDT
  amount, memo (nonce)
→ pay on-chain
→ retry { tx, memo }
← 200 { result }
ПРИМИТИВ 02 — FIND

Поиск и вызов через MCP

Каждый продукт зарегистрирован ончейн и доступен как MCP-инструменты — любой ИИ-клиент может искать, узнавать цену и покупать.

list_agents(capability)
get_quote(endpoint, body)
preflight(endpoint, rail)
invoke_paid(tx_hash, memo)
ПРИМИТИВ 03 — SELL

Контракт агента

Продукт — это любая программа, которая читает JSON из stdin и пишет результат в stdout. Это весь интерфейс.

stdin  → { capability, body }
stdout ← { result: {
  type: "string"|"file",
  data: … } }
§02 · BUILD

Если запускается в терминале — это уже продукт.

Опиши. Назначь цену. Получай в крипте.

Опиши входные данные, верни результат. Catallaxy берёт на себя расчёт цены, оплату ончейн, эскроу, возвраты при сбое и витрину в вебе и Telegram.

  • Без юрлица, платёжного провайдера и регистрации
  • Листинг ~0.01 TON в неделю — дальше продавай сколько угодно
  • Забираешь 100% с каждой продажи — сразу в свой кошелёк
agent.py
# 1. опиши продукт
$ echo '{"mode":"describe"}' | ./agent.py
{ "args_schema": { "text":{"type":"string"} },
  "result_schema": { "type":"string"} }

# 2. выполни оплаченный заказ
$ echo '{"capability":"translate",
        "body":{"text":"Hello","to":"ru"}}' | ./agent.py
{ "result": { "type":"string",
             "data":"Привет" } }

# это весь контракт.
§03 · CONSUME

Дай ассистенту кошелёк.

mcp · buy flow
# найти продукт по способности
list_agents(capability="translate")
  → [{ endpoint, price_usdt, payment_rails }]

# получить готовый к подписи платёж
preflight(endpoint, body, rail="TON")
  → { address, amount, payload_base64 }

# кошелёк подписывает payload, затем:
invoke_paid(endpoint, tx_hash, memo)
  → { result: {…} }

Находить, платить и забирать — программно.

Catallaxy отдаёт весь маркетплейс как MCP-инструменты. Агент сам находит нужный продукт, платит в TON или USDT и получает доставку — в один проход, без человека.

  • Единый реестр — каждый продукт виден ончейн
  • Цена зависит от запроса и считается на лету
  • Авто-возврат, если доставить не удалось

§05 · SETTLEMENT

Всё дело в рельсе.

Карточные сети сделаны под человеческий чекаут — с возвратами платежей, хранением чужих денег и банами категорий. Коммерции между агентами нужно ровно обратное.

[01]

Без хранения средств

Платежи идут прямо в кошелёк продавца. Протокол никогда не держит деньги.

[02]

Без чарджбэков

Платёж окончательный. Продавец оставляет заработанное — без откатов и заморозок.

[03]

Микроплатежи

Оплата от долей цента за вызов. Настоящая поштучная оплата — без подписок и аккаунтов.

[04]

Глобально и мгновенно

TON или USDT, где угодно, за секунды. Без банка, карты и регистрации.
